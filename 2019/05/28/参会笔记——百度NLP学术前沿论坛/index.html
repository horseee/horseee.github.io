参会笔记——百度NLP学术前沿论坛

Posted on 2019-05-28 | In NLP , 学术论坛 | Views:

这两天跑去北京参加了百度的自然语言处理前沿论坛。第一次参与了学术论坛，心情还是比较激动的。总的来说，整体能学到的东西还是很多的，但是就前沿两个字来说，还是和心里的预期有一定的差距。

整体感受

首先先讲一下我听完下来对于技术方面的整体一些新的认知。

  • 关于模型压缩
    很神奇的是，这次会议上，几位学者非常高频次的提及了模型压缩这个方向。在这次会议前，我以为这个方向NLP领域做的人比较少，相关论文也不是很多。孙栩教授在2017年已经做了这个方向的研究，并且现在已经在往模型压缩的可解释性方向做了。后面的一些教授也谈到了未来要做（正在做）模型压缩相关的工作。

  • 关于Bert
    另外在这次会议上，Bert也是一个超级无敌高频词出现的词。很多学者表示了在bert出来了之后，很多搞模型的人都感觉，Bert太牛逼了，再改模型很难再有很大的提升。对于工业界来说是一件非常好的事情，因为很多企业落地AI应用的技术难度直线下降（无论什么任务都只要用Bert就行了），但是对于学术界来说就真的很惨了。现在看起来还唯一没有受到Bert影响的就是机器翻译领域了（因为整体的架构不是端到端的模型并且Bert是一种单语言的架构，和翻译恰恰完全相反）。

  • 未来方向
    阅读理解方向来说，第一个可以做的方向是在知识引入（如何融合知识图谱的信息），第二个是如何做多步的推理。
    整体的NLP领域，其实现在的整体的发展状况还完全不能称之为智能。比如机器阅读理解来说，现在使用的方法本质上还是个相似度计算，所以很容易在小数据集上出现过拟合的问题，机器其实并没有做到能够真正的理解文本。
    另外，NLP现在的主要的方法还是集中于Data-Driven，但是对于人类来说，实际上学习过程是一个无监督学习，只有在考试的时候才会出现有label（答案）的检验。大佬们认为，NLP的下一个阶段，需要着眼于zero-shot或者few-shot的方法，Data-Driven的方法基本上不可能完全解决NLP的问题。
    最后一个也就是可解释性了。这个问题普遍存在于所有的人工智能领域。由于他的不可解释，很多时候我们会把写模型的过程称之为炼丹。

语义计算

从静态到动态的词向量

嘉宾：哈工大 - 车万翔

词向量表示的发展历程

  • one-hot：高维稀疏离散
  • 加权(TF-IDF, PMI)/降维(SVD/LDA): 低维稠密连续
  • Word Embedding
  • 基于上下文的word embedding表示

关于Bert Finetune的论文：To tune or not to tune? adapting pretrained representations to diverse tasks

教授认为，现在一个模型主要是由模型+数据+硬件三个部分组成，可以有以下的几个方向来改善：

  1. 生成伪数据：修改，删除（完形填空），增加（文本顺滑）
  2. 多任务学习

底层表示和顶层模型的关系：
在底层表示足够优秀的情况下，顶层的模型设计更加偏向于完成特定的任务。

车万翔教授的实验室目前研究方向：

  1. 跨语言自然语言处理
  2. 基于Bert的Few-shot
  3. 轻量级的动态词向量

Sparse Deep Learning

嘉宾：北大-孙栩

任务概述：
人类在思考的时候是一种sparse的机制，而现在的神经网络是一种dense的模型，也就是说一旦更新一个batch，需要更新所有的神经元。并且现在神经网络学习所有的特征，偏向于全部表示性，很容易导致过拟合（出租车颜色）

  • meProp
    减少反向传递的时间，部分节点不反向传递。
    论文：meProp: Sparsified Back Propagation for Accelerated Deep Learning with Reduced Overfitting

  • meSimp
    自动化裁剪。

自然语言处理中的表示学习进展

嘉宾：复旦大学-邱锡鹏

Transformer：非常容易overfitting，适合数据量大的任务。
star-transformer：间接连接。在transformer上再额外加上一部分的信息。

论文：
引入先验：https://arxiv.org/abs/1611.03949
How to finetune Bert: https://arxiv.org/abs/1905.05583

XLM: https://github.com/facebookresearch/XLM
MASS: https://arxiv.org/pdf/1905.02450.pdf

教授说他在Bert出来之后，很多优化Bert和Transformer的论文都被拒了。
觉得还可行的方向：

  1. 模型压缩/模型加速
  2. 语句生成（对话）。
  3. Attention其实是动态全连接，是否有可能改进？

百度语义计算技术及其应用

嘉宾：百度-孙宇

语义表示部分百度的发展：

  • 基于term去做word similarity的拓展
  • Topic Model
  • 分布式计算和大规模计算（1T预料，100w超大规模中文词覆盖，RNNLM + hierarchical softmax）
  • ERNIE 知识增强的语义建模。mask实体、短语。

语义匹配百度的发展：

  • SimNet：最初步的匹配框架
  • SimNet2.0：针对各个pairwise做匹配。interaction-based model/Attention-based model
  • IJCAI的一篇论文：主要解决长文本匹配问题，

问答与阅读理解

文本阅读理解的研究进展和挑战

中科院-刘康

  1. bigger seems better。数据集的大小对性能影响很大。
  2. still relevance rather than real understanding。其实很多模型还是在做相似度匹配的工作。
  3. Some questions are too simple
  4. difficult questions need multiple clues
  5. finding clues/evidence is not simple

百度阅读理解技术研究及应用

百度-刘璟

现在NLP的发展，得益于：数据规模变大，人工智能技术的应用。

百度提出的两个模型：
V-NET2018ACL：面向搜索场景的多文档阅读理解任务
思路：每个段落做一个预测答案。在答案之间做对比

KT-NET2019ACL：引入外部知识的阅读理解
引入的知识：知识图谱。
论文：Enhancing Pre-trained Language Representations with Rich Knowledgefor Machine Reading Comprehension（还未公开）

工业角度：

  1. 模型鲁棒性不够：过稳定问题（问题语义改变，但是预测答案不变），过敏感问题（加一个问号就改变答案）
  2. 泛化能力：在A数据集上的模型，能不能很快很好的迁移到其他的数据集上

下午的一些主题都不太了解，我主要以听为主了，这里记得非常简略。

机器翻译&人机对话

机器翻译

这一趴的学术界邀请的是刘洋教授。不愧是ACL的领域主席，确实很多东西看的角度就很高。他们组的工作虽然想法看上去都很容易实现，但是确实我觉得创新性非常的大。
刘洋教授组的可视化神经网络分析的方法（判断每一个输入/每一个神经元在网络中起到的对结果的影响大小），这一个想法感觉可以用来做神经网络的debug，后续打算把这部分的内容仔细看一下，看看能不能对修改网络结构有所帮助。
工业界部分，何中军老师展示了一下最新的百度同传，我们现场体验了一把，感受只能用牛逼两个字来形容。

人机对话

这一部分百度的UNIT负责人孙珂的介绍简直颠覆了我对现在的智能对话系统的认知。孙博士表示，现在百度的对话系统，实际上背后是一种rule-based的机制，而非深度学习的model。原因主要是深度学习的model非常的不经济。原因有以下一些：

  1. 训练的时间成本和性能问题
    孙博士表示，他们尝试了用增强学习的方法来做这个，但是需要训练2000轮才能达到一个还不错的效果。但这个效果仅仅只需要几条rule就能完成
  2. GPU耗费问题
    后来一位教授在panel时提到，有一家公司为了做这个对话系统（深度学习方法），每天的GPU的电费就要上百万。
  3. 稳定
    这里的稳定是指输出结果的稳定。产品最怕的就是对结果的不可控，如果采用深度学习的模型，模型基本是处在一种失控的状态，工程师没法知道产品什么时候会出来什么不太好的东西。

另外，百度的UNIT是一个集成了NLP很多很多很多小的NLP任务的大系统，而非单单仅是靠一个单一的结构就能完成的。
