ERROR-device-side-assert-triggered

Posted on 2019-02-27 | In NLP , PyTorch | Views:

记录一下碰到的两个RuntimeError：device-side assert triggered和backward a second time

device-side assert triggered

在命令前加上CUDA_LAUNCH_BLOCKING=1后，出来了一段新的报错信息。

可以看到，这是一个数组越界的问题。这里要注意的一点是，这里不一定是越了上界，也有可能是出现了负值。基本上来说，device-side assert triggered意味着有数组的越界问题了。
如果是使用jupyter notebook的话，之后所有的在cuda上的操作，无论是多么简单的操作，都会报这个错误（除非shutdown或者换一块卡）。在尝试的过程中，在jupyter notebook中加入os.environ[‘CUDA_LAUNCH_BLOCKING’]=‘1’这一条指令，可能会出不来更具体的报错信息。建议还是使用命令行。

backward througn the graph a second time

这个的错误原因是，因为在backward之后，缓存中存储的数据将被释放。但是有一部分的图被反向传播了两次，导致他需要的数据已经被free掉了。如果按照错误的提示，在backward里面加一个retain_graph = True的话，这样这一部分的内存就不会被释放掉，但是一般我们在实现的时候（除了GAN），都不太会用到这种情况，直接加上这句是错误的。
这个问题比较常出现在RNN中，有一部分的RNN会拿隐藏层去做下一步的输入，也就是${hidden_state}_t = model({hidden_state}_{t-1}, …)$，另外一种情况就是拿最后一个隐藏层作为了下面另外一层的输入，可以看如下代码
